Upload and ingest browser files or folders

Accepts multipart/form-data with one or more file parts and optional relative paths, stages them under storage/raw/uploads//, and ingests through the filesystem connector. Metadata keys filesystem_upload_id, filesystem_upload_root, filesystem_upload_file_count, and filesystem_upload_original_name are added to each result event.

Request Body schema: multipart/form-data
required
files
required
string <binary>

One or more files to upload

paths
string

Relative paths for browser folder uploads (one per file, matching file order)

Responses

https://localhost:8080/filesystem/upload
